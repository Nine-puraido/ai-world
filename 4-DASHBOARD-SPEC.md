```python
def revert_to_checkpoint(task_id: str, checkpoint_id: str):
    checkpoint = db.get_checkpoint(checkpoint_id)

    # 1. If merged, revert the merge commit on main
    if task_was_merged(task_id):
        run(f"git revert --no-commit {merge_commit_sha}", cwd=workspace)
        run(f"git commit -m 'Revert task-{task_id} for rework'", cwd=workspace)

    # 2. Reset task branch to checkpoint's git SHA
    run(f"git checkout task-{task_id}", cwd=workspace)
    run(f"git reset --hard {checkpoint.git_sha}", cwd=workspace)

    # 3. Update DB state
    db.delete_checkpoints_after(task_id, checkpoint.step_index)
    db.delete_events_after(task_id, checkpoint.step_index)
    db.update_task_state(task_id, "EXECUTING")

    # 4. Optionally re-spawn worker from this point
    # Worker gets the conversation history up to this step
    # so it continues from where it was, not from scratch
```

**Three revert options:**
- **Revert** â€” undo steps, leave task paused for manual decision
- **Revert & Re-run** â€” undo steps, spawn new worker to continue from checkpoint
- **Revert Entire Task** â€” undo everything including merge, delete branch
